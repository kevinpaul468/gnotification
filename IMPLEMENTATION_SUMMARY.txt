WHAT WE BUILT:
A production-grade, self-hostable notification service that centralizes all
notification needs (email, SMS, push, etc.) for your organization through a
single API with an extensible plugin architecture.

ARCHITECTURE:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  Your Apps (any language)                                                  │
│      │                                                                      │
│      ├─ POST /notifications/send {provider, recipient, content}            │
│      │                                                                      │
│      ▼                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  Notification Service (REST API - Port 8080)                        │  │
│  │  - Validates requests                                              │  │
│  │  - Stores in PostgreSQL (audit trail)                              │  │
│  │  - Publishes to RabbitMQ queue                                     │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│      │                                                                      │
│      ▼                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  RabbitMQ (Message Queue)                                           │  │
│  │  - "notifications" queue: main queue                                │  │
│  │  - "notifications-retry": retry queue (exponential backoff)         │  │
│  │  - "notifications-dlq": dead letter queue (failed messages)         │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│      │                                                                      │
│      ▼                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  Worker Pool (Process Messages)                                     │  │
│  │  - Consumes from queue                                              │  │
│  │  - Loads provider implementation from registry                      │  │
│  │  - Calls provider.Send()                                            │  │
│  │  - Handles retries (exponential backoff)                            │  │
│  │  - Updates status in DB                                             │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│      │                                                                      │
│      ├─ SUCCESS: Update DB (status: "sent")                                │
│      └─ FAILURE: Retry or move to DLQ                                      │
│                                                                             │
│  Supported Providers (Plugins):                                            │
│  ┌─────────────┬─────────────┬──────────────┐                             │
│  │ SMTP        │ SMS         │ Future:      │                             │
│  │ Gmail       │ Twilio      │ Firebase     │                             │
│  │ Office365   │ AWS SNS     │ Slack        │                             │
│  │ Custom SMTP │ More...     │ Discord      │                             │
│  └─────────────┴─────────────┴──────────────┘                             │
│                                                                             │
│  Database: PostgreSQL (Audit Trail)                                        │
│  ├─ notifications table: all sent notifications                            │
│  ├─ failed_notifications: DLQ messages                                     │
│  └─ provider_configs: provider configurations                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

KEY TECHNOLOGIES:
  Language: Go 1.24.1
    ✓ Single compiled binary
    ✓ Goroutines for concurrency
    ✓ Type safety
    ✓ <1s startup time

  Message Queue: RabbitMQ
    ✓ Persistent queues
    ✓ Dead letter queues (DLQ)
    ✓ At-least-once delivery
    ✓ Exponential backoff retries

  Database: PostgreSQL
    ✓ ACID compliance
    ✓ Audit trail
    ✓ JSONB for flexible config

  HTTP: Echo framework
    ✓ Lightweight
    ✓ Type-safe routing
    ✓ Middleware support

GO PLUGIN ARCHITECTURE EXPLAINED:
  ├─ Define Interface (Provider interface)
  ├─ Implement Interface (one file per provider)
  ├─ Auto-register via init() (no manual setup)
  ├─ Look up from registry (runtime dispatch)
  └─ Result: Type-safe, extensible, compile-time guarantee
      NO dynamic loading like Python, but BETTER type safety!
